Pong 3D/VR con Three.js — Tutorial del código

Volver al juego

Este documento explica, paso a paso, cómo está organizado el proyecto y qué hace cada sección del archivo src/main.js. Es una prueba de concepto con:

  • Modelos 3D de baja fidelidad (mesa, raquetas, pelota, marcador).
  • Controles por teclado y mouse, IA básica para la CPU.
  • Soporte WebXR (VR) con objetos agarrables.

1) Escena, cámara y renderer

Se crea una escena, una cámara en perspectiva y un renderer WebGL con sombras y WebXR.

// Escena y cámara
const scene = new THREE.Scene();
const camera = new THREE.PerspectiveCamera(62, aspect, 0.01, 200);

// Renderer con WebXR
const renderer = new THREE.WebGLRenderer({ antialias: true });
renderer.xr.enabled = true;
  • FOV: cambia el primer valor de PerspectiveCamera(62,...).
  • Sombras/calidad: ajusta renderer.setPixelRatio y dir.shadow.mapSize.

2) Suelo y mesa

La mesa se arma con cajas (tapa/patas), líneas finas blancas y una red simple transparente.

  • Dimensiones: ~2.74 x 1.525 m, altura ~0.76 m.
  • Color/Material: en los materiales de la mesa.

3) Raquetas (paddles) con forma

Cada raqueta es un grupo con:

  • Blade circular delgado (CylinderGeometry) vertical.
  • Mango (BoxGeometry) detrás del blade.
  • Se guarda bladeRadius para colisiones.
function createPaddle(color) {
    const bladeRadius = 0.10;
    const blade = new THREE.Mesh(new THREE.CylinderGeometry(bladeRadius, bladeRadius, 0.02, 32), mat);
    blade.rotation.z = Math.PI / 2; // vertical

}

4) Pelota y física (arcade)

La pelota se mueve en el plano XZ y su altura Y se fija al plano de las raquetas para asegurar colisiones y agarre en VR.

  • Rebotes en Z (bordes de la mesa).
  • Colisión con raquetas: invierte X, añade spin y respeta velocidad máxima.
// Altura fija y avance
ball.position.y = paddleY;
state.ballVelocity.y = 0;
ball.position.addScaledVector(state.ballVelocity, dt);

5) Marcador (CanvasTexture)

Un canvas 2D genera la textura con el score y se asigna a un plano. Se ubica al fondo y siempre se orienta hacia la cámara.

const { texture, update } = createScoreTexture();
const scorePlane = new THREE.Mesh(new THREE.PlaneGeometry(0.8, 0.8), new THREE.MeshBasicMaterial({ map: texture }));
scorePlane.lookAt(camera.position);

6) Controles: mouse, teclado y pausa

  • Mouse: controla el eje Z del paddle del jugador con damping fuerte y feed-forward.
  • Teclado: W/ArrowUp/A = arriba, S/ArrowDown/D = abajo.
  • Pausa: Enter.
  • Clic: bloquea/desbloquea la rotación (paddle erguido).
// Damping y feed-forward
playerPaddle.position.z = THREE.MathUtils.damp(playerPaddle.position.z, targetZ, 80.0, dt);
targetZ += input.mouseYVel * (tableSize.height * 2.6);

7) IA del CPU

Predice la llegada de la pelota y mueve su paddle con velocidad máxima limitada. Se añade una rotación sutil basada en el movimiento.

const move = clamp(delta, -state.cpuMaxSpeed * dt, state.cpuMaxSpeed * dt);
cpuPaddle.position.z += move;
cpuPaddle.rotation.y = THREE.MathUtils.clamp(-dz * 0.6, -0.35, 0.35);

8) VR y objetos agarrables

Se habilita WebXR con VRButton. Se usan raycasters desde los controladores para “tomar” raquetas/pelota. Al agarrar la pelota, su velocidad se pone a 0.

controller.addEventListener('selectstart', onSelectStart);
// ...
newMatrix.decompose(object.position, object.quaternion, object.scale);

9) Bucle de animación

En cada frame se actualizan agarres, jugador, CPU, pelota, cámara y marcador; luego se renderiza.

renderer.setAnimationLoop(() => {
    updateGrabs();
    if (state.running) { updatePlayer(dt); updateCPU(dt); updateBall(dt); }
    updateCameraFP(dt);
    scorePlane.lookAt(camera.position);
    renderer.render(scene, camera);
});

10) Zoom y cámara

El zoom se controla con la rueda del mouse y teclas +/- ajustando state.cameraBack, que se interpola en updateCameraFP para alejar la cámara.

window.addEventListener('wheel', (e) => {
    state.cameraBack = clamp(state.cameraBack + Math.sign(e.deltaY) * 0.08, 0.4, 2.0);
});

// En updateCameraFP
const back = THREE.MathUtils.lerp(0.35, 1.4, state.cameraBack);
camera.position.x = playerPaddle.position.x - back;

¿Qué puedo ajustar rápidamente?

  • Respuesta del paddle: damping (80.0) y feed-forward en updatePlayer.
  • Dificultad CPU: state.cpuMaxSpeed.
  • Pelota: state.maxBallSpeed y spin en colisión.
  • Zoom: rango de state.cameraBack y FOV de la cámara.
  • Marcador: posición del scorePlane y fuente del canvas.
